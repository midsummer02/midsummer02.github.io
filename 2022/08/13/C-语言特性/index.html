C++语言新特性

发表于 2022-08-13 更新于 2022-08-14 分类于 语言学习

本篇博客用于记录 C++ 的语言特性相关知识，比较零碎，一直更新：

C++中的指针和引用：

指针是一个变量，存储的是一个地址，指向内存的一个存储单元，必须分配相应地址空间，同时也可以不初始化，指向NULL，指针也可以有多级，一个指针指向另一个，而且可以修改，++运算是地址的移动；

引用是原变量的一个别名，跟原来的变量实质上是同一个东西，并不分配实际存储空间，但声明时必须初始化，也不能为NULL，不允许有多级定义，也不允许修改，++运算是值的改变。

C++ 程序的内存分配：

1、栈，由编译器在需要时分配，不需要时自动清除的变量的存储区。里面的变量通常是局部变量、函数参数等。

2、堆，就是那些由程序员调用mfreealloc动态分配的内存块，其内存释放与编译器无关，由应用程序控制，一般一个malloc就要对应一个free。如果忘记释放（内存泄漏），在程序结束后，操作系统会自动回收。

3、自由存储区，由new分配的内存块，和堆是十分相似，不过它是用delete 来结束自己的生命的。

4、全局/静态存储区，全局变量和静态变量被分配到同一块内存中，在以前的C语言中，全局变量又分为初始化的和未初始化的，在C++里面没有这个区分了，他们共同占用同一块内存区。

5、常量存储区，存放的是常量，不允许修改，程序结束后系统释放。

6、代码区：存放函数体（类成员函数和全局区）的二进制代码。

new与malloc区别：

1、new内存分配失败时，会抛出bac_alloc异常，它不会返回NULL；malloc分配内存失败时返回NULL。

2、使⽤new操作符申请内存分配时⽆须指定内存块的⼤⼩，⽽malloc则需要显式地指出所需内存的尺⼨。

3、new 和 delete 是 c++ 的运算符， opeartor new /operator delete可以被重载，⽽malloc/free并不允许重载。

4、new/delete会调⽤对象的构造函数/析构函数以完成对象的构造/析构。⽽malloc则不会

5、malloc与free是C++/C语⾔的标准库函数,new/delete是C++的运算符

6、new操作符从⾃由存储区上为对象动态分配内存空间，⽽malloc函数从堆上动态分配内存。

（这里的自由存储区不一定是堆区，也可能是静态区，取决于new操作符在哪里为对象分配内存）

struct和class的区别：

使用 class 时，类中的成员默认都是 private 属性的；而使用 struct 时，结构体中的成员默认都是 public 属性的。class 可以使用模板，而 struct 不能用模板。

限制只在栈或栈上创建对象：

只在栈上：将new运算符重载并设置为私有，使得外部无法访问调用new在堆区中创建对象。

只在堆上：将析构函数设置为私有。原因：静态对象建立在栈上，是由编译器分配和释放内存空间，编译器为对象分配内存空间时，会对类的非静态函数进行检查，即编译器会检查析构函数的访问性。当析构函数设为私有时，编译器创建的对象就无法通过访问析构函数来释放对象的内存空间，因此，编译器不会在栈上为对象分配内存。但是这也意味着delete无法直接调用类的析构函数，需要在类内定义一个 destory() 函数，用来释放new创建的对象。不仅如此。也无法解决继承问题，因为如果这个类作为基类，析构函数要设置成 virtual，然后在派生类中重写该函数，来实现多态。但此时，析构函数是私有的，派生类中无法访问。

构造函数设置为 protected，并提供一个 public 的静态函数来完成构造，而不是在类的外部使用 new 构造；将析构函数设置为 protected。类似于单例模式，也保证了在派生类中能够访问析构函数。通过调用 create() 函数在堆上创建对象。

main函数执行前和后的代码：

全局对象的构造函数会在main 函数之前执行，全局对象的析构函数会在main函数之后执行；

一些全局变量、对象和静态变量、对象的空间分配和赋初值就是在执行main函数之前，而main函数执行完后，还要去执行一些诸如释放空间、释放资源使用权等操作
